# Роль администратора

Администратор — управляющий платформой Storm. У него есть все возможности обычного пользователя плюс административные функции, недоступные остальным.

> Эта и следующие главы описывают **административную специфику**. Общие рабочие функции (проекты, хосты, уязвимости, заметки, отчёты) описаны в документе **Пользователь** — он тоже доступен вам как администратору.

## Что может только администратор

- **Управление пользователями** — приглашать людей в рабочее пространство, назначать роли, редактировать и удалять учётные записи. Раздел **Members** в сайдбаре виден только администраторам (см. «Управление пользователями»).
- **Создание и удаление проектов** — кнопка создания проекта и удаление проекта доступны только администратору (см. «Проекты»).
- **Agent-токены** — токены для API может выпускать любой пользователь в рамках своих прав, но администратор может выдать токен на **любые/все проекты** и **отозвать любой** токен (см. документ **Agent API**).

## Доступ ко всем проектам

Ключевое отличие администратора: он **видит и открывает любой проект без членства в нём**. Обычному пользователю доступны только проекты, где он участник; администратору — все. Это распространяется на все данные проекта: уязвимости, активы, заметки, отчёты, ленту активности.

## Разграничение ролей: две независимые оси

У пользователя две независимые роли:

- **Аккаунтная роль (Workspace role)**: **Admin** или **User** (`pentester`). Определяет административные права на уровне всей платформы.
- **Проектная роль (Project role)**: **Lead** или **Pentester**. Открывает расширенные права **только внутри проектов, где пользователь состоит участником** (управление составом, правка карточки проекта).

Обе роли задаёте вы в разделе **Members**. Не путайте их: Admin — про платформу целиком, Lead — про конкретные проекты пользователя.
