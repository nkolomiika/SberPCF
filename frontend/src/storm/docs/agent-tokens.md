# Токены и права доступа

Agent-токены (в интерфейсе — «API keys») — это секрет, которым автоматизация аутентифицируется в `/api/v2`. Токен может выпустить **любой пользователь**, но строго **в рамках своих прав**.

## Выпуск токена

Токены создаются на вкладке **Profile → API & Automation** (раздел **API keys**, кнопка **Generate new key**). При создании задаются:

- **Имя (Key name)** — человекочитаемая метка (для чего/кому токен), чтобы потом было понятно, что отзывать.
- **Права (Permissions / scopes)** — набор разрешений (см. ниже).
- **Доступ к проектам** — «Все мои проекты» либо явно выбранные проекты.
- **Срок действия** — дата, до которой токен действует; пусто = бессрочный.

Значение токена начинается с префикса **`storm_`** (например, `storm_XXXXXXXX…`).

> **Значение токена показывается ровно один раз — в момент создания.** Storm хранит только его хэш и больше не покажет полный токен. Скопируйте и сохраните его сразу в надёжном месте; если потеряли — выпустите новый.

## Scopes (права)

Права токена определяются набором scopes. Доступны ровно те, что понимает `/api/v2` — ничего лишнего:

- **`projects:read`** — читать список проектов и отдельный проект.
- **`assets:read`** — читать хосты и активы проекта.
- **`vulns:read`** — читать уязвимости.
- **`vulns:write`** — создавать и обновлять уязвимости.
- **`notes:read`** — читать заметки.
- **`notes:write`** — создавать и обновлять заметки.

Токен получает **только выбранные** права — не больше. Неизвестные scopes отклоняются при создании.

> Обратите внимание: **права на запись активов (`assets:write`) нет** — через Agent API активы (хосты/порты/сервисы/эндпоинты) можно только читать. И в v2 **нет удаления** ничего.

## Права токена не превышают ваши

Ключевое правило: **токен не может дать доступ шире, чем есть у вас самих**.

- **Доступ к проектам.** Не-администратор может выдать токену доступ только к тем проектам, где он **сам участник**. «Все мои проекты» для обычного пользователя означает именно его проекты, а не все проекты системы. Администратор видит все проекты, поэтому может выдать доступ к любым/всем.
- **Проверка дважды.** Ограничение проверяется и при создании токена (нельзя выбрать чужой проект), и на каждом запросе к `/api/v2` (доступ токена пересекается с **текущими** правами создателя). Если вы потеряли доступ к проекту, токен тоже его теряет — даже если проект был выбран раньше.

Пример: если у вас нет доступа к проекту №1, вы не сможете выпустить токен, читающий проект №1 — и даже токен с «всеми проектами» не получит к нему доступ.

## Срок действия

Токен может быть **бессрочным** (поле срока пустое) или действовать **до выбранной даты**. По истечении срока запросы с токеном перестают работать. Дату в прошлом задать нельзя.

## Отзыв токена

Ненужный или скомпрометированный токен **удаляется** (кнопка удаления в списке ключей). Вы можете удалять **свои** токены; администратор может удалить любой. После удаления запросы с токеном сразу перестают работать.

## Принцип наименьших привилегий

- Давайте токену **минимально необходимый** набор scopes.
- Ограничивайте токен **конкретными проектами**, если задача касается одного-двух.
- Ставьте **срок действия** для временных задач.
- Отдельные токены — под отдельные интеграции: так проще отозвать один, не задев остальные.
