# Хосты и активы

Активы — это инвентаризация инфраструктуры цели внутри проекта. Работа с ними — во вкладке **Recon** проекта, где переключаются виды **Hosts**, **IPs** и **Endpoints**.

- **Хост** — машина/цель: имя, IP-адреса, статус доступности и его порты.
- **IP-адрес** — адрес, привязанный к хосту (у хоста может быть несколько).
- **Порт** — открытый порт хоста с протоколом и состоянием; на порту может быть обнаружен сервис.
- **Эндпоинт** — HTTP API-точка (метод + путь), удобно для веб/API-тестирования.

## Хосты

Вид **Hosts** показывает список хостов проекта.

**Добавить хост:**
1. Откройте проект → **Recon** → **Hosts**.
2. Нажмите кнопку добавления хоста.
3. Заполните поля:
   - **Hostname** — имя/адрес хоста.
   - **IP address** — основной IP.
   - **Ports** — порты в формате `443/tcp`, вводятся тегами (по Enter на каждый).
   - **Status** — `up` (доступен), `down` (недоступен) или `unknown`.
4. Сохраните.

Хост можно открыть, чтобы посмотреть его IP-адреса, порты и эндпоинты, а также отредактировать или удалить.

## IP-адреса

У хоста может быть несколько IP. Основной задаётся в форме хоста, а дополнительные добавляются в виде **IPs**: там указываются хост, IP-адрес и метка (например, `external` / `internal` / `mgmt`). Все адреса хоста показываются на его карточке.

## Порты и сервисы

Порты добавляются к хосту через поле **Ports** в форме хоста — тегом `порт/протокол` (например, `443/tcp`, `53/udp`).

На карточке хоста порты отображаются таблицей: **порт**, **состояние** (`open` / `filtered` / `closed`) и **обнаруженный сервис**. Состояние и название сервиса показываются, когда эти данные есть (например, из результатов сканирования) — отдельного редактора для них в интерфейсе нет.

## Эндпоинты

Эндпоинты — HTTP API-точки хоста.

**Добавить эндпоинт:**
1. Откройте **Recon** → **Endpoints** (или хост) и нажмите добавление.
2. Укажите:
   - **Host** — к какому хосту относится.
   - **Method** — `GET`, `POST`, `PUT`, `PATCH` или `DELETE`.
   - **Path** — путь, например `/api/users`.
3. Сохраните.

Открыв эндпоинт, можно вставить и сохранить **«сырой» HTTP-запрос** целиком (метод, путь, заголовки, тело — свободным текстом), а также **скопировать запрос как cURL**. Поле ответа в карточке показывается для удобства, но не сохраняется.

Вид **Endpoints** агрегирует эндпоинты по всем хостам проекта — удобно для обзора API-поверхности.

## Экспорт списков

Списки инфраструктуры можно выгрузить (диалог экспорта проекта):

- **Hosts** и **IPs** — как текстовый список (`.txt`).
- **Endpoints** — как текстовый список (`.txt`) или в формате **OpenAPI/Swagger 3** (`.json`).

Выгружается то, что осталось на экране после наложенных фильтров. Про выгрузку Word-отчётов см. главу «Отчёты».
