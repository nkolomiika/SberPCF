# Обзор Agent API

Agent API (версия **v2**) — отдельный программный интерфейс Storm для машинного доступа: автоматизация, CI/CD, сканеры и внешние агенты. Через него внешние инструменты читают данные проектов и загружают в Storm находки и заметки — без интерактивного входа человека.

> Токен может выпустить **любой пользователь** — но только в рамках своих прав (см. «Токены и права доступа»). Поэтому документ **Agent API** доступен всем.

## Чем Agent API отличается от обычного API

- **Отдельный префикс** — машинные запросы идут в `/api/v2` (веб-приложение работает через `/api/v1`).
- **Аутентификация по токену** — доступ определяется **Bearer-токеном** (agent-токеном), а не пользовательскими cookie.
- **Права по scopes и проектам** — что можно сделать токеном, задаётся его **scopes** и списком проектов, к которым он допущен (или флагом «все проекты»).
- **Только чтение и запись, без удаления** — v2 умеет читать, создавать и обновлять; удаления в нём нет.

## Что можно делать

В зависимости от выданных прав токен может:
- читать список проектов и отдельный проект;
- читать хосты и активы проекта;
- читать заметки, а также создавать и обновлять их;
- читать уязвимости, а также создавать и обновлять их.

Записи, сделанные токеном, атрибутируются пользователю, от имени которого выпущен токен (его создателю).

## Как устроена работа

1. Администратор **выпускает токен** с нужными правами и доступом к проектам (см. «Токены и права доступа»).
2. Токен передаётся автоматизации/агенту и хранится как секрет.
3. Агент ходит в `/api/v2`, передавая токен в заголовке `Authorization: Bearer <token>` (см. «Аутентификация и эндпоинты»).
