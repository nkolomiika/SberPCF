# Отчёты

Storm формирует готовые документы Word (`.docx`) на основе реальных данных проекта — хостов, находок, шагов воспроизведения и приложенных доказательств. Отчёт собирается по шаблону; ничего заполнять вручную в Word не нужно.

## Виды отчётов

Доступно два шаблона, заточенных под процесс сертификационных работ:

- **Отчёт для сертификации (СЗИ)** — отчёт по результатам испытаний средства защиты информации, для сертификационных испытаний.
- **Отчёт внутренней приёмки (ПП)** — протокол приёмки, для внутренней приёмки работ.

## Как сгенерировать отчёт

1. Откройте нужный проект.
2. Откройте диалог выгрузки/экспорта проекта.
3. Выберите вид отчёта — **Отчёт для сертификации** или **Отчёт внутренней приёмки**.
4. Запустите генерацию и дождитесь готовности — файл `.docx` скачается автоматически (имя файла формируется из названия проекта и вида отчёта).

Отчёт собирается из данных проекта на момент генерации: система обходит находки, подставляет их поля, разрешает файлы-доказательства по шагам и встраивает изображения в документ. Поэтому перед выгрузкой убедитесь, что находки заполнены: заголовки, CVSS, шаги воспроизведения, влияние и рекомендации, приложены нужные скриншоты.

## Экспорт списков разведки

Из того же диалога, помимо Word-отчёта, можно выгрузить списки инфраструктуры — **Hosts**, **IPs** или **Endpoints**. Выгружается ровно то, что осталось на экране после наложенных фильтров, в одном из форматов: обычный **список** или **OpenAPI** (для эндпоинтов).
