# Работа тимлида

> **Требуется роль Lead.** Возможности из этой главы доступны пользователям с проектной ролью **Lead** — и только внутри проектов, где они состоят участником. Роль лида назначает администратор в разделе **Members**. Те же права есть у **создателя проекта** и у **администратора**.

Роль **Lead** не меняет тип учётной записи — лид остаётся обычным пользователем. Она открывает дополнительные права **в своих проектах**: управлять составом команды и редактировать карточку проекта.

Чтобы права действовали, лид должен быть **участником** этого проекта.

## Управление участниками проекта

Во вкладке **Members** проекта можно **добавлять** и **удалять** участников.

**Добавить участника:**
1. Откройте проект → вкладка **Members**.
2. Нажмите **Add member**.
3. Выберите пользователя из рабочего пространства (начните вводить username).
4. Подтвердите.

Добавленный участник получает **уведомление** о добавлении в проект и сразу получает доступ к данным проекта. Удалить участника можно кнопкой удаления в его строке.

> Заводить новых пользователей рабочего пространства и назначать им роли (в том числе роль Lead) может только администратор в разделе **Members** сайдбара. Лид лишь добавляет в проект **уже существующих** пользователей.

## Редактирование карточки проекта

Лид может редактировать **карточку проекта** во вкладке **Overview** / из строки проекта: название, описание, **статус** и даты начала и окончания.

## Что остаётся только администратору

Даже с ролью Lead нельзя:
- **создавать** и **удалять** проекты;
- заводить пользователей и назначать роли.

Эти операции описаны в документации администратора.
