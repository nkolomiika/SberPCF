# Уязвимости

Уязвимости (находки) — ядро проекта. Каждая находка привязана к хосту и несёт критичность, оценку CVSS, шаги воспроизведения, влияние и рекомендации. Работа с находками — во вкладке **Vulnerabilities** проекта.

## Как завести находку

1. Откройте проект → вкладка **Vulnerabilities**.
2. Нажмите кнопку добавления находки.
3. Заполните быструю форму:
   - **Title** — краткая суть находки.
   - **Affected host** — находка обязана относиться к хосту; выберите его из списка хостов проекта.
   - **Severity** — критичность (см. ниже).
   - **Status** — обычно `open` для новой находки.
4. Сохраните. Откроется карточка находки, где заполняются остальные поля.

> Если нужного хоста ещё нет, сначала заведите его во вкладке **Recon → Hosts** (см. главу «Хосты и активы»).

## Критичность (severity)

Возможные значения: **critical**, **high**, **medium**, **low**, **info**.

У новой находки критичность по умолчанию — **info**. Как только у находки появляется вектор **CVSS**, критичность вычисляется из него автоматически. Чтобы поменять её у находки с CVSS, редактируйте **CVSS-вектор** на карточке.

## Поля карточки находки

- **CWE ID** — тип слабости из каталога CWE (например, `CWE-89`).
- **CVSS 4.0 вектор** — вводится текстом (строка вектора); рядом сразу показываются рассчитанные **балл** и **критичность**.
- **Status** — статус находки (см. ниже).
- **Steps to reproduce** — шаги воспроизведения нумерованным списком: шаги можно добавлять и удалять. В шаг можно вставить скриншот из буфера обмена (открывается по клику).
- **Impact** — влияние.
- **Remediation** — рекомендации по устранению.

## Статусы находки

Статус отражает жизненный цикл уязвимости:

- **Open** — открыта, требует внимания.
- **In progress** — в работе.
- **Fixed** — исправлена.
- **Won't fix** — не будет исправлена.
- **Accepted risk** — риск принят.

Когда статус находки меняется, **автор находки получает уведомление**.

## Фильтры

Список уязвимостей можно фильтровать по **критичности** и **статусу** (мультивыбор), а также по **автору** и **хосту**.
